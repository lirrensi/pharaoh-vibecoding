# INDEX.md Specification

Every folder in `docs/` has an `INDEX.md` file. It is a navigational map — not a content document. It answers: "what's in this folder, and where should I look next?"

---

## Purpose

INDEX.md serves as:
1. **A table of contents** — lists every file and subfolder with one-line summaries.
2. **A search shortcut** — agents and humans scan INDEX.md before opening individual files.
3. **An ontology anchor** — indexes enforce that every doc is discoverable (no orphans).

---

## Format

### Root INDEX.md (`docs/INDEX.md`)

The master map of the entire documentation system.

```markdown
---
node_type: index
updated: 2026-06-09
---

# Documentation Index

## Overview
Brief paragraph about what this documentation system contains and how to navigate it.

## Layers

### overview/ — Product Identity
Why this product exists, who it serves, what it does.
- [product.md](overview/product.md) — Elevator pitch, core capabilities, non-goals
- [personas.md](overview/personas.md) — User roles and journeys
- [INDEX.md](overview/INDEX.md) — Full overview index

### spec/ — Behavioral Specification
What the system must do, exactly, in BDD+RFC format.
- [system-model.md](spec/system-model.md) — Actors, states, core concepts
- [data-model.md](spec/data-model.md) — Entities, schemas, transitions
- [error-handling.md](spec/error-handling.md) — Failure modes, limits
- [INDEX.md](spec/INDEX.md) — Full spec index (+ features/)

### architecture/ — Implementation Structure
How the current implementation realizes the spec.
- [core.md](architecture/core.md) — Shared foundation, boundaries
- [INDEX.md](architecture/INDEX.md) — Full architecture index (+ components/, decisions/)

### Other
- [guides/](guides/INDEX.md) — How-to documents
- [ops/](ops/INDEX.md) — Operational procedures
- [reference/](reference/INDEX.md) — Glossary, conventions
- [changes/](changes/INDEX.md) — Active change proposals
- [archive/](archive/INDEX.md) — Completed change history
- [ontology.md](ontology.md) — This documentation system's ontology

## Status Key
- 🟢 active — Current and authoritative
- 🟡 draft — Work in progress
- 🔴 deprecated — Replaced, kept for history
- ⚫ archived — Completed change, historical

---

*Auto-generated. Last rebuilt: 2026-06-09*
```

### Folder INDEX.md (any subfolder)

```markdown
---
node_type: index
updated: 2026-06-09
---

# {Folder Name} Index

## Documents
- [auth.md](auth.md) — Authentication behavior: login, sessions, 2FA [🟢 active]
- [api.md](api.md) — API gateway behavior: routing, rate limiting [🟡 draft]

## Subfolders
- [features/](features/INDEX.md) — Per-feature behavioral specs

## Tags
`auth` `security` `api` `sessions` `rate-limiting`

---

*Auto-generated. Last rebuilt: 2026-06-09*
```

## Content Rules

### Document Entries
Each document entry follows this pattern:
```
- [filename.md](filename.md) — One-line summary of what this doc covers [status emoji]
```

The summary should:
- Be one line (40-80 chars)
- Describe what the document IS, not just its title
- Include the status emoji from frontmatter

Status emojis:
- `active` → 🟢 (green circle) — or omit if all active
- `draft` → 🟡 (yellow circle)
- `deprecated` → 🔴 (red circle)
- `archived` → ⚫ (black circle)

### Subfolder Entries
```
- [subfolder/](subfolder/INDEX.md) — What this subfolder contains
```

Always link to `INDEX.md`, never just the folder.

### Tags Section
List all unique tags from documents in this folder (comma-separated or space-separated). Helps with cross-cutting search.

## Auto-Generation

INDEX.md files **MUST** be auto-generated by `scripts/index.py`. **NEVER hand-edit an INDEX.md — every hand-edit will be completely wiped and overwritten the next time the script runs.** There is no manual path. There is no fallback. The script is the sole source of truth for INDEX.md content.

The script:

1. Scans every folder under `docs/` recursively.
2. Reads each `.md` file's YAML frontmatter to extract `title`, `node_type`, `status`, and `tags`.
3. Builds the full document link graph and reverse-index so entries show incoming/outgoing links.
4. Generates entries: `- [title](file.name) — status emoji` with graph annotations.
5. Collects all tags into a tag list.
6. **Wipes and overwrites** every `INDEX.md` with the regenerated content.

### Required frontmatter for auto-indexing
- `node_type` (always)
- `title` (for the one-line summary; falls back to filename stem)
- `status` (for the status emoji; defaults to active)
- `tags` (for the tag section)

---

## When to Rebuild

Run `python scripts/index.py` to regenerate all INDEX.md files after:
- Creating a new document
- Updating a document's `title`, `status`, or `tags`
- Moving a document between folders
- Deprecating a document
- Creating or removing a subfolder

**At the end of every session that modified docs, run `python scripts/index.py` to keep INDEX.md current.** Running it when nothing changed is a safe no-op — the script overwrites files but the content is identical if frontmatter hasn't changed.

---

## Quick Validation

These invariants are enforced by `scripts/index.py` — you should not need to validate INDEX.md by hand. A valid INDEX.md:
- Has `node_type: index` in frontmatter
- Ends with `*Auto-generated. Last rebuilt: YYYY-MM-DD*` footer
- Lists every `.md` file in the folder (except itself)
- Lists every subfolder with a link to its `INDEX.md`
- Has summaries derived from frontmatter `title`
- Has status emojis matching frontmatter `status`
- Has a `## Tags` section aggregating all tags from documents in the folder
